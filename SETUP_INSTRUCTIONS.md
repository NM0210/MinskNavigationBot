# Инструкция по настройке .env файла

## Проблема
При запуске бота возникает ошибка: `BOT_TOKEN не найден!`

## Решение

### Шаг 1: Создайте файл .env

Создайте файл с именем `.env` (с точкой в начале) в папке проекта:
```
D:\проект_надя\MinskNavigationBot\.env
```

### Шаг 2: Добавьте токен бота

Откройте файл `.env` в любом текстовом редакторе (Блокнот, Notepad++, VS Code и т.д.) и добавьте следующую строку:

```
BOT_TOKEN=ваш_токен_бота_здесь
```

**Важно:** Замените `ваш_токен_бота_здесь` на реальный токен вашего бота.

### Шаг 3: Как получить токен бота

1. Откройте Telegram
2. Найдите бота [@BotFather](https://t.me/BotFather)
3. Отправьте команду `/newbot`
4. Следуйте инструкциям:
   - Введите имя бота (например: "Минск Навигация")
   - Введите username бота (должен заканчиваться на `bot`, например: `minsk_navigation_bot`)
5. BotFather отправит вам токен, который выглядит примерно так:
   ```
   1234567890:ABCdefGHIjklMNOpqrsTUVwxyz-1234567890
   ```
6. Скопируйте этот токен и вставьте в файл `.env`:
   ```
   BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz-1234567890
   ```

### Шаг 4: Сохраните файл

Убедитесь, что файл сохранен с именем `.env` (не `.env.txt` или другим именем).

### Шаг 5: Запустите бота

После создания файла `.env` с правильным токеном, запустите бота:
```bash
dotnet run
```

## Пример правильного содержимого файла .env

```
BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz-1234567890
```

## Проверка

Если файл создан правильно, при запуске бота вы увидите:
```
✓ Загружен .env файл из: D:\проект_надя\MinskNavigationBot\.env
Бот запущен и готов к работе!
```

## Устранение проблем

**Проблема:** Файл не найден
- Убедитесь, что файл называется именно `.env` (с точкой в начале)
- Убедитесь, что файл находится в папке `MinskNavigationBot`
- Проверьте, что файл не сохранен как `.env.txt`

**Проблема:** Токен не читается
- Убедитесь, что в файле нет лишних пробелов
- Формат должен быть: `BOT_TOKEN=токен` (без кавычек)
- Убедитесь, что токен скопирован полностью
